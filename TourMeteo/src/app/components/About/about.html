  • Recherche multi-villes : comparer la météo horaire de plusieurs villes sur une même date
  • Filtres d'affichage : température, vent, résumé météo (activables/désactivables)
  • Vue mobile optimisée : cartes avec emoji météo en fond, indicateur jour/nuit, précipitations

Importez un fichier .gpx via la page GPX. L'application parse les points trkpt, calcule la distance totale (formule de Haversine), puis estime l'heure de passage à chaque point selon la vitesse moyenne renseignée.

Pour chaque point, un reverse-géocodage Nominatim identifie la ville, puis la météo horaire est récupérée via Open-Meteo. Les doublons consécutifs sont filtrés automatiquement.

Le tracé est affiché sur une carte interactive (Leaflet) avec des marqueurs numérotés.

Un score de sortie vélo (0-100) est calculé en fonction de la température, du vent, des précipitations et des conditions spéciales. Des recommandations de tenue et des conseils sont générés.

Les résultats peuvent être exportés en image PNG ou partagés via l'API Web Share.

Les données sont obtenues en temps réel depuis les APIs publiques listées en section développeur.

Les coordonnées GPS (latitude/longitude) sont envoyées aux services tiers suivants : Nominatim, Open-Meteo, BRouter, OSRM, Overpass, Open Elevation et CARTO (tuiles). Aucune autre donnée personnelle n'est transmise.

Nominatim applique des limites d'usage (1 requête/seconde). L'application utilise un throttle et un échantillonnage par distance (~2 km) pour respecter ces contraintes.

Aucun compte utilisateur, aucun cookie de tracking, aucune donnée personnelle stockée côté serveur. Les préférences et l'historique restent en localStorage sur votre appareil.

  • Géocodage : Nominatim (OpenStreetMap, ODbL) — recherche et reverse-géocodage. Politique : 1 req/s max.
  • Météo : Open-Meteo (CC BY 4.0) — données horaires (température, ressenti, vent, précipitations, humidité, weathercode).

Services complémentaires (expérimental) :

  • Routage : BRouter (MIT) / OSRM (BSD)
  • Analyse voirie : Overpass API (OSM, ODbL)
  • Élévation : Open Elevation / fallback Open-Meteo
  • Tuiles : CARTO Voyager (CC BY 3.0) · OpenStreetMap (ODbL)

Aucune clé API n'est requise. Tous les services sont gratuits et open-source.

  • GpxUploaderComponent : orchestrateur principal (upload, parsing GPX, calcul d'itinéraire, récupération météo)
  • GpxMapComponent : carte Leaflet avec tracé polyline et marqueurs numérotés
  • RideScoreComponent : calcul du score de sortie vélo, tenue recommandée, alertes
  • GpxSummaryBarComponent : barre de statistiques résumées (distance, durée, départ, arrivée)
  • GpxResultsTableComponent : tableau desktop et cartes mobile des passages météo
  • GpxExportService : export Canvas → PNG et partage Web Share
  • HistoryService : sauvegarde/chargement des trajets en localStorage
  • FeatureFlagService : feature flags persistées en localStorage
  • WeatherSheetComponent : grille météo horaire par ville (desktop table + mobile cards)
  • SearchTabComponent : formulaire de recherche et filtres
  • WeatherUtils : weathercode → emoji, degrés → direction cardinale
